# Random Bible Verse Generator - 设计思路文档

## 一、项目定位

### 核心定位
**简洁高效的随机圣经经文生成器 + 图片分享平台**

### 目标用户
1. **主要用户**：日常灵修的基督徒
2. **次要用户**：需要灵感指引的普通用户
3. **潜在用户**：社交媒体分享者

### 差异化策略
- ✅ **极简界面**：专注核心功能，减少干扰
- ✅ **图片生成**：一键生成精美图片，便于分享
- ✅ **多版本支持**：支持多种圣经译本
- ✅ **主题分类**：按需筛选相关主题
- ✅ **移动优先**：响应式设计，随时随地使用

---

## 二、功能设计

### 核心功能（MVP）

#### 1. 随机经文生成
```
功能描述：
- 点击按钮随机生成一条圣经经文
- 显示经文内容、章节出处
- 支持快速刷新获取新经文

技术实现：
- 前端：随机算法选择经文
- 数据：JSON 格式存储圣经数据
- 交互：无刷新加载（AJAX）
```

#### 2. 图片生成与分享
```
功能描述：
- 将经文转换为精美图片
- 自定义样式（背景、文字、字体）
- 一键分享到社交媒体

技术实现：
- html2canvas 或 Canvas API
- 预设模板 + 自定义选项
- 社交分享 API（Facebook, Twitter, WhatsApp）
```

#### 3. 主题筛选
```
功能描述：
- 20+ 主题分类（Love, Hope, Faith, Peace 等）
- 按主题生成相关经文
- 主题切换不影响用户体验

技术实现：
- 经文标签系统
- 前端筛选 + 后端查询
- URL 参数支持（/love, /hope）
```

#### 4. 多版本支持
```
功能描述：
- 支持 3-5 种主流译本（NIV, ESV, KJV, 中文和合本）
- 版本切换实时更新
- 版本对比功能（可选）

技术实现：
- 多版本数据存储
- 前端切换逻辑
- 本地存储用户偏好
```

### 扩展功能（V2.0）

#### 5. 每日经文
```
- 基于日期的固定经文
- 每日推送提醒（可选）
- 历史记录查看
```

#### 6. 收藏功能
```
- 收藏喜欢的经文
- 收藏列表管理
- 导出收藏（文本/图片）
```

#### 7. 搜索功能
```
- 关键词搜索
- 按书卷搜索
- 按章节搜索
```

## 三、技术架构

### 前端技术栈

```
核心框架：
- React / Vue.js（推荐 React，生态丰富）
- Next.js（SSR + SEO 优化）

样式方案：
- Tailwind CSS（快速开发）
- 或 CSS Modules（组件化样式）

状态管理：
- React: Zustand / Redux Toolkit
- Vue: Pinia

工具库：
- html2canvas（图片生成）
- react-share（社交分享）
- framer-motion（动画效果）
```

### 后端技术栈

```
方案一：无后端（纯前端）
- 静态站点（Vercel/Netlify）
- 数据存储在 JSON 文件
- 适合 MVP 快速上线

方案二：轻量后端
- Node.js + Express
- 或 Python + FastAPI
- 数据库：SQLite（小项目）/ PostgreSQL（扩展）

方案三：Serverless
- Vercel Functions / AWS Lambda
- 数据库：Supabase / Firebase
```

### 数据存储

```
圣经数据：
- 格式：JSON
- 结构：
  {
    "book": "John",
    "chapter": 3,
    "verse": 16,
    "text": "For God so loved...",
    "version": "NIV",
    "topics": ["love", "salvation", "faith"]
  }

用户数据（可选）：
- 收藏列表（LocalStorage）
- 偏好设置（LocalStorage）
- 历史记录（LocalStorage）
```

## 五、用户体验流程

### 主流程

```
1. 用户访问网站
   ↓
2. 看到简洁界面 + 大按钮
   ↓
3. 点击"生成随机经文"
   ↓
4. 显示经文（带加载动画）
   ↓
5. 用户可以选择：
   - 再次生成
   - 生成图片
   - 切换主题
   - 切换版本
   ↓
6. 如果生成图片：
   - 打开自定义弹窗
   - 调整样式
   - 下载或分享
```

### 关键交互细节

```
1. 加载状态
   - 按钮点击后显示 loading
   - 经文淡入动画
   - 平滑过渡

2. 错误处理
   - 网络错误提示

3. 响应式设计
   - 移动端：单列布局
```

## 九、技术难点与解决方案

### 难点 1: 圣经数据获取

```
问题：需要完整的圣经文本数据
解决方案：
1. 使用开源数据（如 Bible API）
2. 手动整理（工作量较大）
3. 购买商业数据（成本较高）

推荐：使用开源 API
- api.bible
- bible-api.com
- 或 GitHub 开源数据
```

### 难点 2: 图片生成性能

```
问题：html2canvas 可能较慢
解决方案：
1. 使用 Canvas API 直接绘制（更快）
2. 预生成常用模板
3. 服务端生成（Serverless Function）
4. 缓存生成的图片

推荐：Canvas API + 缓存策略
```

### 难点 3: 多语言支持

```
问题：需要多语言界面和数据
解决方案：
1. i18n 库（react-i18next）
2. 语言切换组件
3. 多语言数据存储

推荐：先做英文，后续扩展
```
